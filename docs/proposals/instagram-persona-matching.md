# 📸 인스타 페르소나 매칭 — 기획서

> **한 줄 요약**: 사용자의 인스타그램(공개) 미감을 AI로 분석해 **"촬영 페르소나"** 를 만들고, 그에 어울리는 **samae 사진·작가를 추천·연결**하는 유입/전환/바이럴 기능.

| 항목 | 내용 |
|---|---|
| 문서 상태 | v0.2 (PoC 검증 반영, 2026-07-08) |
| 작성 | samae 팀 |
| 관련 서비스 | samae 본서비스 (사진으로 작가를 고르는 매칭 마켓플레이스) |
| 선행 리서치 | 인스타 데이터 확보 기술/법률 현황 (별도 리서치 완료) |
| PoC | §15 참고 — 실제 게시물 분석 + 실 API 데이터 shape 검증 완료 |

---

## 1. 배경 & 문제 정의

### 1-1. 배경
- samae의 핵심 가치: **"마음에 드는 사진을 고르면, 그 작가에게 바로 문의."** 사진이 곧 작가 선택 기준.
- 그러나 신규 사용자의 **콜드스타트 문제**: 처음 온 사용자는 "내가 뭘 원하는지, 어떤 무드가 나한테 맞는지" 모른 채 방대한 포트폴리오를 헤맴 → 이탈.
- 동시에 **유입 훅 부재**: samae를 알릴 강력한 바이럴 매개가 없음. (인스타 광고 컷툰은 인지, 하지만 "직접 해보고 싶은" 인터랙션은 없음)

### 1-2. 문제
1. 신규 사용자가 **자기 취향을 언어화/시각화하지 못해** 첫 사진 선택까지의 마찰이 큼.
2. samae를 **처음 접하게 만드는 재미있는 진입점**이 없음.
3. 인스타그램이라는 슈퍼앱과의 **차별점**이 약함 (인스타도 사진을 보여줌).

### 1-3. 기회
- "내 인스타 분석해줘" 는 검증된 바이럴 포맷 (My Insta Personality, Roast 등 다수 성행).
- 기존 인스타 분석 앱들은 **분석 후 끝**(휘발성). samae는 **분석 → 어울리는 사진 → 작가 예약**이라는 **전환 깔때기가 이미 존재**.
- 즉 같은 훅으로 유입시켜도 samae만 **매출로 연결**할 수 있음 → 구조적 우위.

## 2. 목표 & 성공지표

### 2-1. 목표
| 구분 | 목표 |
|---|---|
| 유입 | 공유 가능한 결과물로 신규 방문 유입 (바이럴 계수 ↑) |
| 활성화 | 신규의 첫 사진 탐색/문의까지의 마찰 감소 (콜드스타트 해결) |
| 차별화 | "인스타는 남의 사진, samae는 네 사진" 포지셔닝 확보 |

### 2-2. 성공지표 (KPI)
- **북극성**: 분석 완료 → **작가 문의 전환율** (analysis→inquiry CVR)
- 보조지표
  - 분석 완료율 (username 입력 → 결과 도달)
  - 공유율 (결과카드 공유/저장 비율)
  - 공유 → 신규 유입 (viral K-factor)
  - 추천 사진 클릭률 / 추천 작가 문의율
  - 재방문·재분석률
- **가드레일**: 건당 원가(스크래핑+AI) ≤ 목표치, 스크래핑 실패율, 사용자 불만/정확도 체감

## 3. 타겟 사용자 & 유즈케이스

### 3-1. 타겟
- **1차**: 촬영을 고민 중이지만 취향이 막연한 20~30대 (프로필/커플/여행 스냅 수요).
- **2차**: 재미로 "내 인스타 분석"을 해보려는 인스타 헤비유저 (바이럴 유입).

### 3-2. 대표 유즈케이스
1. *"곧 커플 촬영하고 싶은데 어떤 느낌으로 할지 모르겠어"* → 인스타 분석 → "너희는 필름 감성 자연광 커플" + 예시 사진 + 매칭 작가.
2. *"인스타 분석 재밌겠다"* (재미 유입) → 결과카드 공유 → 친구도 유입 → 그중 일부가 실제 촬영 문의.
3. *(기존 samae 유저)* 탐색이 막막할 때 → 분석으로 시작점(무드) 잡고 추천 피드로 진입.

## 4. 핵심 컨셉 & 차별점

### 4-1. 컨셉: "촬영 페르소나"
단순 성격분석이 아니라 **"당신을 찍는다면 이런 사진"** 이라는 **촬영 관점의 정체성**을 부여.
- 예: `필름 감성 도심 산책러` / `자연광 밝은 데이리 무드` / `무드있는 다크 시네마틱`

### 4-2. 차별점
- **인스타 분석 앱**: 분석 → 끝. samae: 분석 → **어울리는 실제 사진 → 실제 작가 → 예약**.
- **인스타(슈퍼앱)**: 남의 피드 구경. samae: **네가 찍힐 사진**을 제안하고 실현.
- **바이럴 카피**: *"인스타는 남의 사진, samae는 네 사진."*

### 4-3. 확장성 (비전, MVP 범위 아님)
- 커플이 서로 태그해서 "우리 커플 촬영 페르소나", 친구 그룹 무드, 시즌별 무드 추천, 작가가 "이 페르소나 전문" 뱃지, 페르소나 기반 추천 피드 개인화 등. → **끝없이 확장 가능하나 MVP는 한 바퀴만.**

## 5. 사용자 플로우 (상세)

```
① 진입 (인스타 광고/공유카드/앱 내 배너)
      ↓
② username 입력 화면
   - "@인스타 아이디로 내 촬영 페르소나 알아보기"
   - 공개계정 안내 + 동의 체크(본인 공개정보 분석 목적)
      ↓
③ 분석 로딩 (5~15초, 재미 요소: "피드 톤 읽는 중…", 진행 애니)
   - [백엔드] 공개 데이터 수집 → 비전/텍스트 분석 → samae 태그 매핑
      ↓
④ 결과 화면 (핵심)
   - 촬영 페르소나 라벨 + 짧은 서술 (근거 문구 포함: "네 피드의 따뜻한 톤·자연광 기반")
   - 무드/유형/로케이션/대표컬러 요약
   - "너한테 어울리는 사진" (실제 samae 포트폴리오 6~9장)
   - "이 작가랑 잘 맞아" (매칭 작가 2~3명, @핸들·대표컷)
   - [공유하기] 결과카드 저장/공유 CTA
      ↓
⑤ 전환
   - 추천 사진 탭 → samae 사진 상세 → 기존 [무료로 상담 신청하기] 플로우
   - 또는 매칭 작가 → 작가 프로필 → 문의
      ↓
⑥ (실패 fallback) 스크래핑 실패/비공개 계정
   - "사진 3~5장 직접 올리기" 로 우회 → 동일 분석 파이프라인
```

## 6. 기능 상세 스펙

### 6-1. 입력
- 인스타 username 1개 (본인, 공개계정 전제).
- 동의: "본인 공개 인스타 정보를 촬영 취향 분석 목적으로 사용" 체크. (제3자/팔로잉 분석 없음 명시)
- fallback 입력: 사진 3~5장 직접 업로드.

### 6-2. 데이터 수집
- **수집 범위 (최소·본인 공개데이터만)**: 프로필사진, bio, 최근 게시물 이미지 9~12장 + 각 캡션.
- **미수집(명시적 제외)**: 팔로워/팔로잉 명단, 타인 게시물, 좋아요/조회수, DM. → 법·약관 리스크 회피.
- **수집 방식**: 스크래핑 벤더 API (§8-2) 또는 fallback 업로드.
- **캐시**: 동일 username 24~72시간 캐시(원가 절감 + 재분석 UX).

### 6-3. 분석 엔진 (2트랙)
**A. 비전 분석** (게시물 이미지 → 시각 신호)
- 색/보정: 따뜻·차가움·뉴트럴 / 필름·디지털 / 채도·명도(밝음↔다크 무드)
- 피사체: 인물·풍경·음식·사물·펫·셀피 비중
- 인물 구성: 혼자 / 커플 / 그룹 / 가족
- 배경·씬: 도심 · 자연 · 카페/실내 · 여행 · 스튜디오
- 스타일: 미니멀 · 빈티지 · 스트릿 · 클래식
- 구도: 클로즈업 · 전신 · 캔디드 스냅 · 정적 포즈

**B. 텍스트 분석** (bio + 캡션)
- 관심사 키워드, 톤(유머·감성·시크·활발), 이모지/해시태그 성향.

→ 두 트랙을 합쳐 **정규화된 특성 벡터** 생성.

### 6-4. 🔑 번역 레이어 — 인스타 신호 → samae 태그 매핑
(추천 품질의 핵심. samae 카탈로그 태그 체계와 1:1로 이어져야 함)

| 인스타 신호 | → samae 태그(예시) |
|---|---|
| 따뜻한 톤 + 필름 그레인 + 낮은 채도 | **무드: 감성 필름** |
| 밝은 노출 + 자연광 + 야외 | **무드: 자연광 밝은 / 데일리** |
| 어두운 톤 + 도심 야경 + 고대비 | **무드: 무드있는 다크 / 시네마틱** |
| 미니멀 구성 + 뉴트럴 톤 | **무드: 미니멀 클린** |
| 인물 클로즈업 + 셀피 다수 | **유형: 프로필/개인** |
| 커플 2인 구성 반복 | **유형: 커플** |
| 친구 그룹 다수 | **유형: 우정/그룹** |
| 여행지·해외 배경 빈도 | **로케이션: 여행 / 야외** |
| 카페·실내 빈도 | **로케이션: 카페·실내** |
| 도심 거리 빈도 | **로케이션: 도심 스냅** |

> ※ samae 포트폴리오·작가가 **동일 태그 체계로 사전 태깅**돼 있어야 매핑이 성립 (§8-4 선행작업).

### 6-5. 촬영 페르소나 생성
- 특성 벡터 → **대표 무드 라벨 1개 + 보조 무드** + 짧은 서술(2~3문장) + 어울리는 유형/로케이션 + 대표 컬러 팔레트.
- **정확도 체감 규칙**: 반드시 **실제 근거 문구** 포함. (예: *"네 피드의 78%가 따뜻한 자연광·필름 톤이라, 밝고 부드러운 무드가 어울려요."*) → 바넘 + 실증의 조합.

### 6-6. 추천
- **사진 추천**: 페르소나 태그와 매칭되는 samae 포트폴리오 사진 6~9장 (태그 유사도 기반).
- **작가 추천**: 작가별 대표 무드/유형 태그와 코사인 유사도 → 상위 2~3명, @핸들·대표컷·한 줄 이유.
- 각 추천에 **"왜 어울리는지"** 한 줄 근거 (설명가능성 = 신뢰).

### 6-7. 공유 카드 (성장 엔진)
- 세로 이미지(인스타 스토리 규격 1080×1920) 자동 생성.
- 구성: 페르소나 라벨 + 대표 컬러 + 어울리는 사진 미리보기 + **samae 로고/워터마크 + "내 촬영 페르소나 알아보기" CTA URL**.
- 브랜드: samae 다크 톤 + 레드 포인트(#ff3d2e), Fraunces 워드마크.
- 저장/공유(인스타 스토리·카톡) 원터치.

### 6-8. 전환
- 추천 사진 → 기존 **사진 상세 → [무료로 상담 신청하기]** 플로우 재사용.
- 매칭 작가 → 작가 프로필 → 문의.
- 비로그인 사용자도 결과까진 보고, 문의 시 로그인(카카오/이메일) 유도.

## 7. 데이터 & 기술 아키텍처

### 7-1. 파이프라인
```
[username] → 수집(스크래핑 벤더/업로드)
          → 이미지 9~12장 + bio/캡션
          → 비전 분석(멀티모달 LLM) + 텍스트 분석
          → 특성 벡터 → samae 태그 매핑
          → 페르소나 생성(LLM) + 추천(태그 유사도)
          → 결과 JSON + 공유카드 렌더
          → [Supabase 저장/캐시]
```

### 7-2. 스크래핑 벤더 (후보 비교)
| 벤더 | 제공 | 대략 단가 | 비고 |
|---|---|---|---|
| **Apify Instagram Scraper** | 공개 프로필·게시물·캡션 | 게시물 $1.5/1천 수준 | 대중적, 안정적, MVP 적합 |
| EnsembleData | 프로필·게시물 | Bronze $200/월~ | 실시간, 무료 50/일 |
| Bright Data | 프로필·게시물 등 | 성공요청 $0.75/1천 | 대규모용, 고급 |
| **직접 업로드(fallback)** | 사용자 제공 | $0 | 벤더 장애·비공개 대비 필수 |

- **MVP 선택**: Apify(주) + 직접 업로드(fallback). 벤더 종속·장애 대비해 **추상화 레이어**로 감싸 교체 가능하게.

### 7-3. 비전/LLM
- 게시물 이미지 분석: 멀티모달 LLM(최신 Claude 등)로 톤/피사체/씬 라벨링.
- 페르소나·추천 서술: LLM + samae 태그 컨텍스트 주입.
- (선택) 이미지 임베딩 기반 유사도 매칭 병행 검토.

### 7-4. 🔑 선행작업 — samae 카탈로그 태깅
- **추천이 "진짜"가 되려면 포트폴리오/작가가 동일 태그 체계로 태깅돼 있어야 함.**
- 기존 무드 태그가 있으면 재활용, 없으면:
  - 포트폴리오 사진 자동 태깅(비전 배치) → 검수.
  - 작가별 대표 무드/유형 프로필 태그 산출.
- **이게 없으면 이 기능은 장난감에 그침 → 최우선 선행.**

### 7-5. 저장
- Supabase: 분석 요청/결과, 캐시, (동의 기반) 재분석용 최소 저장. 원본 이미지 장기저장은 지양(개인정보 최소화).

## 8. 개인정보 · 법률 · 정책

- **범위 최소화**: 본인 공개계정 + 본인 동의. **팔로잉/타인 데이터 절대 미수집** (GDPR·개인정보법 고위험 회피).
- **포지션**: "본인 공개정보, 촬영 취향 매칭·재미 목적."
- **저장 최소화**: 분석 결과 위주, 원본 이미지 즉시 폐기/단기캐시. 삭제 요청 대응.
- **약관 고지**: 인스타 데이터 수집 방식·목적·보관기간 명시, 동의 획득.
- **국가별**: EU 사용자는 리스크 커서 초기엔 국내 한정 권장.
- **스크래핑 리스크 인지**: 인스타 차단·벤더 장애 가능 → fallback 필수, 벤더 계약상 책임 확인.

## 15. PoC 결과 & 검증 (2026-07-08)

브라우저(로그인 세션) 기반 수동 PoC로 **① 분석 파이프라인 유효성**과 **② 실제 API 데이터 shape**를 검증. (수집 방식은 PoC=수동, 프로덕션=벤더/OAuth로 다름 — §15-4 참고)

### 15-1. 검증 A — 분석이 계정을 뚜렷이 구분하는가? ✅
동일 파이프라인(게시물 이미지 → 비전 분석 → samae 태그 → 페르소나)을 두 계정에 적용:

| 대상 | 도출 페르소나 | 주 태그 |
|---|---|---|
| @samae_photo_official (브랜드 몽타주) | "필름 감성 자연광 스냅" | 감성필름 0.82 · 커플 0.70 · 자연/야외 0.68 |
| kimjazz99 (개인, 실데이터) | "바이럴 카페·디저트 크리에이터 / 애니·개발자" | 카페·활동 스냅 · 따뜻 데일리 · (커플/감성필름 아님) |

→ **결론**: 계정마다 확연히 다른 축으로 분리됨. 바넘(누구에게나 맞는 말)이 아니라 **실제 콘텐츠 신호 기반**임을 확인. 단, 정확도 체감을 위해 "실증 근거 문구" 강제 규칙은 유지 필요(§6-5).

### 15-2. 검증 B — 코드로 실제 뽑히는 데이터 (실 API 응답) ✅
Instagram 내부 엔드포인트(스크래핑 벤더가 감싸는 바로 그 API)를 실제 호출해 확인한 **획득 가능 데이터**:

**프로필** (`/api/v1/users/web_profile_info`)
- username, id, full_name, biography, bio_links, external_url
- followers / following / posts 카운트, is_private, is_verified, is_professional, has_clips, profile_pic(HD URL)

**게시물** (`/api/v1/feed/user/{id}`) — **건당 125개 필드**
- code, 미디어종류(이미지/영상/캐러셀), **like_count · comment_count · play_count(조회수)**
- 캡션 전문·해시태그, 게시일시(taken_at)
- **원본 이미지 URL · 영상 URL**(원해상도 + 썸네일 다중 사이즈)
- **인스타 자동 alt텍스트**("Photo by … May be an image of …") — 무료 시각 신호
- 위치, 태그된 사용자, 캐러셀 장수, 음악·릴스(clips) 메타데이터, 원본 크기

→ **결론**: 이미지/영상 URL이 나오므로 → **그 URL을 vision API에 넣는 프로덕션 분석 경로**가 성립. 즉 "서비스로 구현해도 동일 데이터로 동일 분석" 성립.

### 15-3. PoC가 드러낸 실전 이슈 (스펙 반영 필요)
1. **`web_profile_info`는 이제 게시물 0개 반환** → 게시물은 `feed/user/{id}` 별도 호출 필요(+user_id 선확보). 스펙: **2단계 수집**.
2. **페이지네이션** — feed는 1회 ~12건(next_max_id). 전체 수집 시 페이징 로직 필요.
3. **릴스 비중 매우 큼**(개인 계정 예: 9/12) → 영상 분석은 **커버프레임만 vs 프레임 추출** 결정 필요(원가·품질 트레이드오프). §6-3에 "영상 처리 정책" 추가.
4. **자동 alt텍스트를 신호로 활용** — 인스타가 제공하는 공짜 이미지 설명, 비전 비용 절감에 활용 가능(§6-3 보강).
5. **좋아요/댓글 "누가"는 카운트만** 옴 — 개별 신원 미획득(추가 호출 + 제한 + 법 리스크). 스펙상 **미사용 확정**.
6. **OCR보다 API가 정확** — PoC 중 bio를 스크린샷은 "살자", API는 "살래"로 반환. **텍스트는 반드시 API 원문 사용**.

### 15-4. ⚠️ 아직 검증 안 된 것 (다음 PoC)
- **본 PoC는 로그인 세션 기반 수동 호출.** 프로덕션의 **익명·대규모 수집 안정성/단가/차단율은 미검증.** → **Apify 등 벤더 실계정으로 별도 검증 필요** (건당 원가·성공률·릴스 처리).
- 카탈로그(포트폴리오·작가) 태깅 선행작업 미착수 → 추천 매칭 품질은 아직 미검증.

### 15-5. PoC 종합 판단
- **분석 유효성 = 검증됨** (계정 구분·실데이터 기반).
- **데이터 획득 가능성 = 데이터 shape 검증됨** (이미지/영상 URL + 풍부한 메타).
- **남은 리스크 = 순수하게 "익명 대규모 수집의 안정성·비용·법"** → 벤더가 해결하는 영역, 계약·단가·차단율 실측이 다음 관문.
