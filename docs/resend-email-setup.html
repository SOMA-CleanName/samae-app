samae.ai · 이메일 인증

Resend로 회원가입 인증 메일 보내기

DNS는 Vercel · 인증은 Supabase Auth · 발송은 Resend SMTP

목표 — Supabase Auth가 보내는 메일(회원가입 인증·매직링크·비밀번호 재설정)을 noreply@samae.ai 주소로, Resend를 통해 안정적으로 발송. 코드 변경 없이 대시보드·DNS 설정만으로 끝납니다.

1Resend에 도메인 추가

발신 도메인을 등록하고, Vercel에 넣을 DNS 레코드를 받습니다.

  1. resend.com 가입/로그인
  2. Domains → Add Domain → samae.ai 입력 (Region은 기본 us-east-1 무방)
  3. 추가하면 DNS 레코드 목록이 표시됩니다. 값은 도메인마다 다르니 이 화면을 켜둔 채 다음 단계로.

2Vercel에 DNS 레코드 추가

Vercel Dashboard → Domains → samae.ai → DNS Records → Add

아래 4개를 추가합니다. Name(이름)에는 samae.ai를 붙이지 마세요 — Vercel은 입력한 이름 뒤에 자동으로 도메인을 붙입니다.

TypeNameValue비고
MXsend feedback-smtp.us-east-1.amazonses.com Priority 10
TXT
SPF		send v=spf1 include:amazonses.com ~all 발신 인증
TXT
DKIM		resend._domainkey p=MIGfMA0GCSq…
(Resend가 준 긴 키 그대로 복사)		 서명
TXT
DMARC		_dmarc v=DMARC1; p=none; 정책(권장)
주의 · Region이 us-east-1이 아니면 MX·SPF의 us-east-1 부분이 다르게 표시됩니다 — Resend 화면 값을 그대로 복사하세요. 값 끝에 점(.)이 붙어 있으면 빼고 넣어도 됩니다.
이미 apex(@)에 다른 SPF TXT가 있어도 괜찮습니다. 위 SPF는 send 서브도메인용이라 충돌하지 않아요.

3도메인 인증(Verify)

  1. Resend로 돌아가 Verify 클릭 (전파에 몇 분~수십 분)
  2. 모든 레코드가 초록색 Verified 가 되면 완료
팁 · 안 잡히면 Vercel 레코드의 Name이 정확한지(send, resend._domainkey, _dmarc), DKIM 키가 잘렸는지 확인하세요.

4Resend API 키 발급

이 키가 SMTP 비밀번호로 쓰입니다.

  1. API Keys → Create API Key
  2. 이름 supabase-smtp, 권한 Sending access
  3. re_… 키 복사 — 한 번만 표시되니 안전히 보관

5Supabase 커스텀 SMTP 연결

Supabase Dashboard → Authentication → Emails → SMTP Settings → Enable Custom SMTP 켜고 입력

Sender email
noreply@samae.ai
Sender name
사매
Host
smtp.resend.com
Port
465 (안 되면 587)
Username
resend
Password
re_… (4단계 API 키)
Sender email의 @ 뒤 도메인은 반드시 Resend에서 Verified된 도메인(samae.ai)이어야 합니다. 아니면 발송이 거부돼요.

저장 후 Authentication → URL Configuration에서 Site URL이 https://samae.ai 인지 한 번 더 확인.

6테스트

  1. 시크릿 창에서 이메일로 신규 가입 → 인증 메일 수신 확인
  2. 안 오면 Resend → Logs에서 발송/실패 사유 확인
  3. 스팸함도 확인 (초기 p=none 단계라 평판 쌓이기 전엔 스팸 가능)
자주 막히는 원인 — ① 도메인 Verified 전 ② Sender 도메인 불일치 ③ DKIM 키 잘림 ④ 포트(465↔587).
