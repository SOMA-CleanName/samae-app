## 6. 상담 정보(인테이크) — 채팅방 (추가 기능)

채팅 시작(예약·문의) 시 고객이 촬영 정보를 입력하고, 작가가 채팅방에서 수시로 열람.

- **모델**: `consultation_briefs`(대화별 1건, PK=`conversation_id`) — 성별·인원·목적·희망일정·희망지역·자유메모 + `ref_image_paths text[]`(레퍼런스 최대 5장). **마이그레이션 `0015`** ⚠️ 원격 DB 적용 필요 — `node scripts/migrate.cjs 0015`. (※ `budget_krw` 컬럼은 0015에 있으나 UI에서 제외 — 사용 안 함, 정리 시 drop 가능)
- **RLS**: 열람=대화 참여자(고객+해당 작가), 작성/수정=고객만. (실제 쓰기는 service_role 라우트, RLS는 방어선)
- **표시 방식**: 모달은 **선택** — 고객이 미작성 상태로 첫 진입하면 자동 1회 오픈(건너뛰기 가능), 헤더 `📋 상담 정보` 버튼으로 언제든 작성/수정. 작가는 같은 버튼으로 **읽기 전용 열람**.
- **레퍼런스 사진**: `samae-chat` 버킷 `briefs/{conversationId}/...`(공개 읽기 재사용), 서버 라우트 `POST /api/brief`가 sharp 리사이즈·업로드·업서트. 제거된 이미지는 스토리지에서 정리. 최초 작성 시 작가에게 시스템 메시지로 알림.
- **구현**: `BriefPanel`(client, 작가 열람/고객 폼), `lib/chat.getBrief`, 채팅 `page.tsx` 헤더에 패널 배치.

## 7. 채팅 리스트 개선

- **실제 대화 방만 노출**: `last_message_at`이 있는(메시지/시스템 메시지가 한 번이라도 오간) 방만 리스트에 표시 — 예약·문의로 생성만 되고 비어 있는 방은 숨김.
- **채팅방 나가기**: per-side 숨김(`user_hidden_at`/`photographer_hidden_at`). **마이그레이션 `0016`** ⚠️ 원격 DB 적용 필요 — `node scripts/migrate.cjs 0016`. `leaveConversation` 액션이 내 쪽 숨김 시각+안읽음0 기록. 새 메시지가 오면(`last_message_at > hidden_at`) 다시 표시.
- **진행 상태 배지**: 예약 단계에서 파생 — 촬영 완료(shot/delivered/completed) · 예약 완료(accepted/paid) · 상담 중(그 외). `listChatRooms`가 예약 상태를 (user:photographer) 쌍으로 집계.
- **구현**: `lib/chat.listChatRooms`/`ChatStatus`, `LeaveButton`(client, 확인 후 나가기), 리스트 `page.tsx` 재구성(상태 배지 + 나가기). 안읽음 배지 집계용 `listConversations`는 전체 유지.
