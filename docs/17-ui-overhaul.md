# 17. UI/UX 전면 개편 (2026-06, 멘토링 기반)

> 배경: 6/24 멘토링(이한솔)에서 UI/UX 문제 진단 → 6/25 내부 전면 개편 회의.
> 한 줄 원칙: **"사용자는 인내심이 없다 — 거슬리는 건 전부 빼고, 홀린 듯 10~15초 안에 연락처를 남기게."**
> 최우선 플로우: **광고 진입 → 사진 상세 → (채팅형) 문의 전환.** 전환 이후엔 사용자에게 뭘 요구해도 됨(로그인/앱설치 모달 등).
> 전제: **완전 비로그인 기반.** 장바구니·최근 본·문의 등 전부 localStorage. 로그인 기능은 현재 빠진 상태.

---

## A. 전역 (모든 화면 공통)

- **A1** 현행 하단 메뉴바 **삭제**
- **A2** 플로팅 내비: 화면 하단 **중앙**에 `홈`/`탐색` pill(border-radius) 버튼 2개, 작게 떠 있음
- **A3** 플로팅 **장바구니**: 화면 하단 **우측**, 담은 사진들이 겹친 카드 스택으로 **모서리만 살짝** 노출
  - 클릭 → 카드가 슉 펼쳐지며 **장바구니 모달**
  - 닫으면 원위치로 **빨려들어가며 접힘**
- **A4** 알림(알람) 제거
- **A5** 좋아요 **전면 제거** → `+`(장바구니 담기)로 일원화
- **A6** 작가 **실명 노출 전면 금지** (사진 카드·상세·프로필 어디에도). 프로필 페이지는 유지하되 이름 대신 **"사진작가"** 표기 또는 생략 → 이름 검색으로 개인 계정 추적 방지
- **A7** 별점·후기 UI **전역 숨김** (데이터·기능은 보존, 노출만 off)
- **A8** 공유 기능 — **단위: 사진** (필수)
- **A9** 다크모드 (필수)
- **A10** 개인정보 동의 약관 + 동의 플로우 (필수)
- **A11** **혜택형 글로벌 CTA(훅)**: 사이트 방문 시부터 떠 있음(혜택 카피 예 "지금 상담신청하고 할인받기"). 거기서 **예약 또는 장바구니**를 1회 누르면 이후 안 뜸(localStorage). 이후엔 사진 상세의 예약 버튼으로 진입

## B. 홈(메인) 화면

- **B1** 핀터레스트식 메이슨리 — 심심하지 않은 배치로 다듬기 (현 prefix-stable 메이슨리 기반)
- **B2** 사진 카드: 작가 이름 X · 좋아요 X · `+`(장바구니 담기) 버튼
- **B3** ⭐ **담기 모션**: 담는 순간 사진이 하단 우측 장바구니로 **빨려들어가는** 애니메이션 (핵심 — 저장됐음을 각인 + 언제든 열람)
- **B4** 사진 보고 돌아와도 배치 유지 (이미 구현됨: prefix-stable)
- **B5** 광고 진입 온보딩 스포트라이트 (현행 유지, 추후 더 나은 방식 리서치)

---

## C. 탐색 화면 (신규 — 메인과 별도 페이지)

- **C1** 메인과 다른 라우트, `탐색` 버튼으로 진입
- **C2** 상단 검색바
- **C3** 그 아래 **카테고리 섹션 세로 나열** — 각 섹션 = 카테고리명 + 사진 몇 개 **가로 스크롤 미리보기**
- **C4** 카테고리 **세분화 다수** — 광고용 4개와 **별개 체계**. 기존 태그 기반으로 임의 분류(초안은 개발이 제안)
- **C5** 섹션 클릭 → 메인과 동일 레이아웃 + 그 카테고리 사진 **상단 우선** (`fetchCategoryFeed` 재사용)

## D. 사진 상세 화면

- **D1** 작가 **이름 전부 노출 금지**
- **D2** "작가 페이지 버튼"에서도 이름 빼기 (버튼은 유지, 익명)
- **D3** 하단 **"작가 사진" 탭 제거** → 스크롤 내리면 **추천 사진만**
- **D4** 별점·후기 숨김
- **D5** 예약/문의 CTA **최상단**으로
- **D6** ⭐ **CTA 카피/버튼명 변경** (hook 연결·혜택형). "예약·문의하기"는 누르기 싫음 → 욕구 자극형
- **D7** 캐러셀 "휘리릭" 모션 완화/개선
- **D8** blur→선명 진입 모션 제거
- **D9** 상단 nav: 평소 숨김 → 위로 스크롤 시 노출
- **D10** 상세 진입 시 검색바 안 뜨게
- **D11** 이탈 경로(딴 데로 튀는 곳) 차단
- **D12** `+`(장바구니 담기)

## E. 문의 = 채팅방 형식 (가장 중요)

기존 한 페이지 폼 → **스크롤되는 가짜 채팅방**.

진입: ① 혜택형 글로벌 CTA(A11) 또는 ② 사진 상세 예약 버튼(D6) → 둘 다 같은 채팅 플로우.

흐름:
1. 채팅방 진입 → 시스템이 먼저 **진입한 그 사진**을 말풍선으로 보냄
2. **0.5초 딜레이** 후 시스템 **질문 말풍선** (타이핑 인디케이터)
3. 딜레이 후 **사용자 쪽 말풍선**에 **답변 버튼들** 나열 → 선택
4. 선택 시 다음 질문으로, 스크롤로 쌓임
5. 질문 순서: 목적 → 날짜 → 지역 → 인원수 → 추가내용 → **연락처 + 개인정보 동의**
6. **제출 전까진 이전 답변 언제든 수정 가능**
7. 제출 = 리드 전환

규칙:
- **선택형은 전부 버튼**, 자유입력은 **연락처만 텍스트**. 웬만하면 다 버튼.
- 각 질문에 **소프트 스킵 버튼**: "모르겠어요 / 아직 못 정했어요 / 작가님과 상의 후 정할게요"
- **언제든 연락처만 바로 입력 → 전부 스킵** 가능한 경로 상시 제공
- 목표: **10~15초 안에 연락처**
