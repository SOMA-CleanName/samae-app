# 08. 기능 백로그 (추가·수정 결정 사항)

회의에서 추가/수정하기로 결정한 기능을 영역별로 정리한다. 각 항목은 **무엇을·왜·결정 필요 지점**을 명시한다.
미정 사항은 ❓, 베타 제외는 🚫(beta), 결정 완료는 ✅로 표기.

> 기존 흐름은 [01 제품 개요](01-product-overview.md) · [05 예약 라이프사이클](05-booking-lifecycle.md) · [06 결제·정산](06-payment-settlement.md) · [07 로드맵](07-roadmap.md) 참조.

## 1. 사진 (업로드)

- **업로드 = "피드"(인스타식 게시물)** ✅ (구현 완료 2026-06-04)
  - **최종 모델**: 업로더 하나(`PortfolioUploader`)로 1장이든 N장이든 선택 → **같이 올린 사진 = 하나의 피드**(`createPost`로 album 생성, 각 사진에 `album_id`). 선택적 공통정보(가격·장소·무드·공개).
  - **공개 프로필**: 피드 **대표 1장만** 그리드에 (여러 장은 ⧉), 클릭 시 **스와이프 캐러셀**(`PhotoCarousel`, 대표가 1/N 첫 장).
  - **탐색 탭**: 묶음 무시하고 **낱개 사진을 랜덤**으로 노출(`shuffle`). 클릭하면 그 사진의 피드 캐러셀로.
  - **스튜디오 포트폴리오**: **피드별 그룹**으로 묶어 관리(사진별 공개/편집/삭제).
  - 용어: "묶음" → **"피드"**.
- **가격 정보 기입** ✅ (구현 완료 2026-06-04) — 사진 단위로 메타 정보 부착.
  - 기입 항목: **가격, 장소만** (패키지 정보는 사진에 붙이지 않음)
  - **가격 정보 온/오프 토글** — 작가가 가격 노출 여부를 선택.

**결정·구현 내용 (가격·장소):**
- 마이그레이션 `0008` — photos에 `price_krw`(정수, null 허용)·`location_text`(촬영 장소, 지역 `region`과 별개) 추가. *(price_visible 컬럼도 추가됐으나 아래 모델 변경으로 현재 미사용 — 추후 0009에서 제거 가능.)*
- 작가 편집 = 포트폴리오 카드별 **모달**(`PhotoMetaEditor`)에서 **가격·장소만** 입력. 서버 액션 `updatePhotoMeta`.
- **가격 온/오프 = 작가가 아니라 "보는 사람" 토글** (결정 2026-06-04). 탐색 메인 상단 "가격 보기" 토글(`PriceToggle`, `?price=1`)을 켜면 갤러리 카드에 가격(설정된 사진만) 노출, 끄면 숨김.
- 작가 프로필 썸네일에는 가격·장소가 설정돼 있으면 표시(작가측 노출 게이팅 없음).

## 2. 패키지

- 패키지 개념은 유지하되 **사진에는 직접 붙이지 않음** (사진 = 가격·장소만).
- 패키지 정보는 **작가 홈 3단계**에서 노출 (아래 5번 참조).

## 3. 예약 플로우

### 3.1 진입 버튼 ✅ (구현 완료 2026-06-04)
- 문의+예약을 **단일 "예약·문의하기" 버튼**으로 통합 (모든 페이지: 작가 프로필·사진 상세).
- 버튼 → **채팅방이 바로 열림**(`startConversation`).

### 3.2 작가별 예약 템플릿 ✅ (구현 완료 2026-06-04, 채팅 일원화)
- **결정**: Q1 표준항목+안내문+출장비 / Q2 `/book` 폐기(채팅 일원화) / Q3 수락=체결(accepted)까지.
- 작가는 스튜디오 **"예약 설정"** 에서 **예약 안내문 + 출장비** 템플릿 작성(`0011`: `photographers.booking_note`·`travel_fee_krw`).
- 채팅 헤더 **"예약하기"**(고객) → 템플릿 모달(패키지·희망시간·장소·메모·**출장비 옵션**) → **제안하기**.
- 제안 = `bookings`(requested) 생성 + 채팅에 **예약 카드 메시지**(`messages.booking_id`). 작가 **수락** → `accepted` 체결 + 시스템 메시지.
- 변경: `BookingComposer`·`ChatRoom` 예약카드·`proposeBooking`, `createBooking`/`/book` 제거.
- ⚠️ 미검증: 전체 제안→수락 플로우는 두 계정(고객/작가) 수동 테스트 권장.

### 3.3 예약·환불 정책 (기본값)
- **무조건 선입금.**
- **촬영 3일 전까지만 취소 가능**, 그 이후에는 **완전히 환불 불가**.
- **작가 협의 조건(예외):** 천재지변 시 작가가 **환불 또는 날짜 미루기** 가능.
  - ❓ **천재지변 조건의 정의** — 무엇을 천재지변으로 인정할지 기준 필요.
- **예약 미루기 / 취소 기능** 추가 필요.

### 3.4 수수료 회수 시점
- 수수료는 **사진 전달 시점까지** 받는다.
  - 근거: 노쇼가 나도 거래는 "종료"로 처리되므로, 전달 완료를 기준으로 정산.
- 플랫폼 수수료 모델은 [결제 모델: 직접 계좌이체 + 건당 6천원] 정책과 연동.

### 3.5 사진 AS(보정 재요청)
- **사진 AS 요청 메시지** 기능.
- ❓ **AS 처리 주체** — 사진작가가 직접 할지, 플랫폼(우리)이 중개할지 확인 필요.

## 4. 작가 일정·예약 관리 ✅ (구현 완료 2026-06-04)

- **작가 일정 관리**: **시간 + 장소** 단위로 관리.
- **작가 측 예약 관리 시스템**: **달력 뷰부터** 우선 구현.

**결정·구현 내용:**
- **결정**: 슬롯 길이=패키지 소요시간(Q1=3) / 차단 시점=수락 시(Q2=1) / 화면=주간규칙+차단+월달력(Q3=1).
- 모델 전환: 개별 슬롯(`availability`) → **주간 반복 규칙(`availability_rules`) + 특정시간 차단(`availability_blocks`)** (`0012`). 예약은 `shoot_at`+`duration_min` 저장.
- 작가 **일정 관리** 화면: **드래그/클릭 시간표 격자**(`WeeklyGrid`, 1시간 칸, 연속 칸 구간 병합) + 특정시간 차단 + 이번 달 예약 달력 + 다가오는 예약.
- 고객 예약: **날짜 선택 → 그 요일 규칙 기반 빈 시간**(차단·기존예약 제외, `lib/slots.ts`) 선택.
- 수락 시 **시간 충돌 검사** → 겹치면 거부, 체결되면 그 시간 자동 차단.
- 시간은 KST 기준 처리.

## 5. 작가 홈 (사진 클릭 → 진입 화면) ✅ (구현 완료 2026-06-04)

사진을 누르면 **사진 상세(`/photos/[id]`)** 로 진입.

**결정 (2026-06-04):**
- Q1 전환 방식 / Q2 라우트 → 처음엔 1·2·3 단계 탭이었으나, **최종적으로 단계 탭 폐기**.
- Q3 "좋아요 수" → **좋아요=찜 통합** (신규 기능 없이 "찜 수"만 표시). 공개 집계는 `0010` SECURITY DEFINER 함수.
- Q4 → 사진 정보는 그 사진 중심(가격·태그·위치·찜수). Q5 → **랜덤 추천 포함**(같은 무드 우선).

**최종 구조 (단일 화면):**
- 사진 상세 = **한 화면**: 사진 크게 + 옆/아래에 가격·태그·위치·찜수·문의/예약·랜덤추천.
- 정보 영역에 **작가 미니 프로필 카드** → 클릭 시 **작가 상세 프로필(`/photographers/[handle]`)** 로 이동.
- 🚫(beta) 유사한 무드 사진 추천(자동 큐레이션)은 제외, 대신 랜덤 추천으로 대체.
- 변경: `photos/[id]/page.tsx` 신설, 탐색 홈 `PhotoCard` 클릭 → `/photos/[id]`.

## 6. 홈 화면 필터링 ✅ (구현 완료 2026-06-04)

- **검색어(태그) 관련 결과만** 노출.
- **검색칸(검색바)만** 남기고 **나머지 필터 UI는 전부 제거.**

**결정·구현 내용:**
- 무드 카테고리 탭 + 지역 칩 **제거** (좌측 네비 레일·메이슨리 갤러리는 유지).
- 검색 = **무드 태그 부분 일치** → **사진(메이슨리) 결과** (기존 작가 카드 검색 폐기).
- 검색어 없을 때 = **전체 published 사진 최신순** (현행 유지).
- 변경: `page.tsx`(탭/칩 제거), `discovery.ts`(`searchPhotosByTag` 추가, `fetchFilterOptions`·`searchPhotographers` 제거), `TopBar.tsx`(플레이스홀더 문구).
- ⚠️ 베타 한정: 태그 부분 일치는 최근 published 200장 범위 내 JS 필터. 규모 커지면 DB 인덱스/검색 함수로 전환 필요.
