# 09. 개발 진행 계획 (항목별 + 질문 루프)

[08 기능 백로그](08-feature-backlog.md)를 **하나씩 차례로** 구현하기 위한 진행 계획.
핵심 원칙: **착수 전에 애매한 부분을 질문으로 먼저 확정한 뒤** 코드를 짠다. "일단 만들고 보기"를 지양한다.

---

## 작업 루프 (항목마다 반복)

각 백로그 항목은 아래 5단계를 거친다.

1. **🔍 현황 확인** — 관련 코드/스키마가 이미 있는지 파악, 재사용 지점 찾기.
2. **❓ 질문 → 확정** — 아래 "착수 전 질문"을 사용자에게 묻고 결정 기록. (가정으로 진행하지 않음)
3. **🛠 구현** — 단일 책임으로 분리해 작성, 주석은 한국어.
4. **✅ 검증** — 동작 확인(필요 시 `/run`·`/verify`), 완료 기준 충족 확인.
5. **📝 문서 갱신** — [08](08-feature-backlog.md)의 해당 항목·미정 표(Q1~Q5) 갱신, 결정 사항 기록.

> 질문은 **착수 시점에 그 항목 것만** 묶어서 묻는다. 한 번에 다 묻지 않고, 차례가 온 항목 위주로.

---

## 현재까지 구축된 것 (기준선)

골격은 존재. 이번 백로그는 대부분 **기존 화면의 보강/수정**이다.

- 인증: 로그인 (`(auth)/login`)
- 작가 스튜디오: 신청·패키지·포트폴리오·가능시간·프로필·정산 (`(photographer)/studio/*`)
- 유저: 탐색 홈·찜·채팅·작가 프로필·예약·결제 (`(user)/*`)
- 어드민: 작가 승인 (`(admin)/admin/*`)
- 스키마: `0001`~`0007` (init→상태가드→포폴스토리지→채팅→예약슬롯→결제정산→직접이체)

## 항목별 "착수 전 질문" (예고)

차례가 오면 묻게 될 핵심 질문들. 미리 생각해 둘 수 있게 나열한다.

### 1. 홈 필터링 단순화 §6
- 현재 탐색 홈에 있는 좌측 레일/필터를 **완전히 제거**? 아니면 검색바로 흡수?
- 검색은 **태그만** 대상인가, 이름/지역도 포함?
- 빈 검색(미입력) 시 기본 노출은 무엇? (전체 랜덤 / 최신 / 인기)

### 2. 사진 가격·장소 기입 §1
- 가격 입력 단위/통화 표기 (원, 1,000 단위)?
- 장소는 **자유 텍스트** vs 지역 선택(드롭다운)?
- 가격 온오프 시 숨김 표시를 어떻게? ("가격 비공개" 라벨 / 아예 미표시)
- 기존 `photos` 테이블에 컬럼 추가 → 마이그레이션 `0008` 필요. 확정?

### 3. 사진 묶음 저장·업로드 §1
- "묶음"의 정의 — 앨범(영구 그룹) vs 한 번의 업로드 배치?
- 묶음에 **공통 가격·장소**를 한 번에 부여? 아니면 장당 개별?
- 묶음도 포트폴리오 그리드에서 하나로 보이나, 펼쳐 보이나?

### 4. 작가 홈 단계별 구조 §5
- 1→2→3단계 전환 방식 — 스크롤 / 탭 / 단계 버튼?
- 2단계 **포트폴리오 노출 여부** (백로그 Q3) 확정 필요.
- "찜한 사람 수·좋아요 수" — 좋아요는 **신규 기능**인가? (현재 찜만 존재) 추가?
- "랜덤 추천"의 대상·범위 (같은 무드 / 전체 작가)?

### 5. 문의/예약 버튼 §3.1
- 문의 = 채팅방 생성으로 바로 연결? 예약 = 예약 화면?
- 비로그인 클릭 시 로그인 유도 흐름?

### 6. 예약 정책·환불·미루기/취소 §3.2~3.4
- 천재지변 조건 정의 (백로그 Q1) — 어떤 사유를 인정?
- "촬영 3일 전" 기준 시각 (촬영일 00:00 기준? 시간 단위?)
- 미루기: 횟수 제한? 새 날짜는 작가 가능시간에서만?
- 환불 불가 구간의 **수수료(건당 6천원)** 처리 — 노쇼 시에도 청구?
- 직접 계좌이체 모델에서 "선입금" 확인을 누가/어떻게 검증?

### 7. 사진 AS 요청 §3.5
- AS 처리 주체 (백로그 Q2) — 작가 / 플랫폼?
- AS 요청 가능 기간·횟수 제한?
- 채팅 메시지로 보낼지, 별도 요청 상태값을 둘지?

### 8. 작가 일정·예약 관리(달력) §4
- 기존 `availability`(가능시간)와 새 "일정 관리"의 관계 — 통합 vs 별도?
- 달력에 표시할 것 — 가능시간 + 확정 예약 + 장소?
- 월/주/일 뷰 중 베타 범위?

### 9. UI 정리 §7
- 정훈이 리서치 결과(스타일·레퍼런스) 확정 대기 (백로그 Q5).
- `ui-ux-pro-max` 스킬로 후보 제안받을지?
