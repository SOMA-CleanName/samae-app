# 11. 유저 플로우 (역할·케이스별)

samae 서비스의 모든 사용 흐름을 **사용자(고객)** · **작가** 관점에서 케이스별로 정리한다.
근거: 실제 코드(서버 액션·RLS·라우트). 예약 상태 전이는 §D 상태 머신 참조.

> 관련: [08 백로그](08-feature-backlog.md) · [09 개발계획](09-dev-plan.md) · [10 리비전](10-revision-feedback.md)

---

## 0. 역할 & 권한

| 역할 | 판별 | 진입 영역 |
|------|------|-----------|
| **게스트(비로그인)** | `getCurrentUser()===null` | 탐색·사진·작가 프로필 열람만 |
| **사용자(고객)** | `role='user'`, `photographer=null` | 탐색·찜·채팅·예약 |
| **작가** | `photographers` 행 보유 | 위 + `/studio`. `status`: `pending`(승인 대기) → `approved` |
| **운영자(admin)** | `role='admin'` | `/admin` (작가 승인·환불 처리) |

- 비로그인이 보호 동작(좋아요·예약·채팅) 시도 → `/login?next=…` 로 유도 후 복귀.
- 미승인(`pending`) 작가의 사진/프로필은 RLS로 **탐색·공개 노출에서 자동 제외**.

---

## A. 사용자(고객) 플로우 — 케이스별

### A1. 탐색·발견 (`/`)
- 메이슨리 갤러리(승인 작가 published 사진, 랜덤). 좌측 레일 + 상단 검색바.
- **가격 보기 토글**: 사진 재로딩 없이 가격만 on/off(클라이언트).
- 무드 태그 검색 → 결과 갤러리.
- 사이드바 배지: **예약(진행 중 수)** · **채팅(안읽음 수)**.

### A2. 사진 상세 (`/photos/[id]`)
- 배경=페이지 기본색, 사진 비율에 맞춰 정보 영역 가변.
- **좋아요(♥)** + 좋아요 수 (관심 작가와 별개 기능).
- 작가 미니 카드 → 작가 프로필 이동("작가 사진 보러가기").
- 하단 탭: **추천(전체 랜덤 무한 스크롤)** ↔ **작가 포트폴리오**.
- CTA: **예약·문의하기** → 게스트면 로그인, 본인 사진이면 스튜디오, 그 외 채팅 시작.

### A3. 작가 프로필 (`/photographers/[id]`)
- 2열 레이아웃: 좌측 정보(아바타·지표·소개·태그·패키지) + 우측 포트폴리오.
- **관심 작가 추가/해제**(♡/♥).
- 포트폴리오 클릭: **데스크톱=모달**(사진 좌·정보 우, 좋아요·예약 가능) / **모바일=사진 상세 페이지**(+뒤로가기).
- CTA: **예약·문의하기**(하단 고정/사이드바).

### A4. 찜 (`/favorites`)
- 2탭: **좋아요한 사진** / **관심 작가**.

### A5. 문의 시작 → 채팅 (`/chat/[id]`)
- "예약·문의하기" → 대화방 get-or-create 후 진입.
- **첫 진입 시 상담 정보 모달 자동 오픈(선택, 건너뛰기 가능)**:
  성별·인원·목적·희망일정·희망지역·자유메모 + **레퍼런스 사진(최대 5장)**.
  → 저장 시 작가에게 시스템 메시지+알림. 헤더 `📋 상담 정보`로 언제든 수정.
- 메시지: 텍스트, **사진 보내기(+ 메뉴)**, 카카오톡식 시간 표시.

### A6. 예약 제안 (헤더 `📋 예약 제안`)
- 패키지 선택 + 희망 일시(또는 협의) + 장소 + 출장비 여부 + 메모 → 제안.
- 금액 = 패키지가 + (출장비). 대화에 **예약 카드** 생성, 상태 `requested`.
- **수정/취소**: `requested` 상태에서만 예약 카드의 수정·취소 버튼.

### A7. 작가 수락 후 — 송금 (직접 계좌이체)
- 작가가 수락 → 상태 `accepted`("송금 대기"), 알림 수신.
- 예약 상세에서 **송금 안내 보기**(`/bookings/[id]/pay`) → 작가 계좌로 직접 이체.
- 작가가 입금 확인하면 `paid`.
- ※ 플랫폼은 **매칭 건당 6,000원 수수료(작가 부담)**. 고객은 패키지 금액만 작가에게 송금.

### A8. 촬영·전달·완료
- `paid` → 작가 **촬영 완료**(`shot`) → **보정본 전달**(`delivered`).
- 고객이 **전달 확인 · 거래 완료**(`completed`).

### A9. 취소·환불
- `requested`/`accepted`: **예약 취소**(양측 가능) → `cancelled`.
- `paid`/`shot`/`delivered`: 고객(또는 운영자) **환불 요청** → `refunded`.

### A10. 예약 기록 (`/bookings`)
- **확정(accepted 이상) 예약만** 목록 표시. 요청중/거절/취소는 **채팅방에서만** 확인.

### A11. 채팅 리스트 관리 (`/chat`)
- **메시지가 오간 방만** 노출(빈 방 숨김). 행에 **상태 배지**(상담중/예약완료/촬영완료).
- **나가기**: 내 쪽에서만 숨김 → 새 메시지 오면 다시 표시.

## B. 작가 플로우 — 케이스별

### B1. 작가 신청 → 승인 대기
- `/studio/apply` 에서 `applyAsPhotographer` → `photographers` 행 `status='pending'`.
- 운영자 승인 전까지 공개 노출 안 됨. 승인되면 `approved`.

### B2. 스튜디오 (`/studio`) 관리
- **포트폴리오**: 사진 업로드(앨범/단건, 가격·장소·무드), 게시/숨김.
- **패키지**: 이름·설명·가격·시간·보정본 수.
- **가용 시간**: 주간 규칙(`availability_rules`) + 차단(`blocks`). 수락 시 충돌 검사 근거.
- **프로필**: 소개·지역·무드·출장비·예약 안내문(`booking_note`).

### B3. 문의 응대
- 고객이 채팅 시작 → 작가에게 새 메시지 알림.
- 헤더 `📋 상담 정보`로 **고객 상담 정보 수시 열람**(읽기 전용, 레퍼런스 사진 포함).

### B4. 예약 요청 처리 (`requested`)
- 채팅 예약 카드 또는 예약 상세에서 **수락/거절**.
- **수락 시 시간 충돌 검사**: 점유 예약과 겹치면 거부("해당 시간에 이미 예약").
- 수락 → `accepted`(체결, 슬롯 점유) / 거절 → `rejected`.

### B5. 입금 확인 (`accepted` → `paid`)
- 고객 송금 후 **입금 확인**(`confirmTransfer`) → `paid`.

### B6. 촬영·전달 (`paid` → `shot` → `delivered`)
- **촬영 완료 표시** → `shot`. **보정본 전달 표시** → `delivered`.
- 이후 고객의 완료 확인 대기(`completed`).

### B7. 취소·환불·정산
- 결제 전(`requested`/`accepted`) 작가도 취소 가능.
- 환불은 고객/운영자 주도. **매칭 수수료 6,000원(작가 부담)** 은 정산에서 집계(`platform_fees`).

## C. 작가 신청·승인 (운영자 포함)

| 단계 | 행위자 | 결과 |
|------|--------|------|
| 신청 | 사용자 | `photographers.status='pending'` |
| 승인/반려 | 운영자(`/admin`) | `approved` → 공개 노출 / 반려 |
| 활동 | 작가 | 스튜디오·예약 수락 가능 |

---

## D. 예약 상태 머신 (핵심)

```
requested ──(작가 수락)──▶ accepted ──(작가 입금확인)──▶ paid ──(작가)──▶ shot ──(작가)──▶ delivered ──(고객 확인)──▶ completed
    │                          │                                   └──────────────┬───────────────┘
    │(작가 거절)               │(양측 취소)                                      (고객/운영자 환불)
    ▼                          ▼                                                   ▼
 rejected                  cancelled                                           refunded
```

| 현재 상태 | 액션 | 행위자 | 다음 상태 | 알림/메시지 |
|-----------|------|--------|-----------|-------------|
| (없음) | proposeBooking | 고객 | `requested` | 예약 카드 "📋 예약을 제안했어요" |
| requested | updateBooking | 고객(본인) | requested | "✏️ 예약 제안을 수정했어요" |
| requested | acceptBooking | 작가 | `accepted` | "✅ 예약이 수락되어 체결" + 충돌검사 |
| requested | rejectBooking | 작가 | `rejected` | "❌ 거절되었어요" |
| requested/accepted | cancelBooking | 고객 또는 작가 | `cancelled` | 상대에게 알림 |
| accepted | confirmTransfer | 작가 | `paid` | 입금 확인 |
| paid | markShot | 작가 | `shot` | 촬영 완료 |
| shot | markDelivered | 작가 | `delivered` | 보정본 전달 |
| delivered | confirmCompletion | 고객 | `completed` | 거래 완료 |
| paid/shot/delivered | refundBooking | 고객/운영자 | `refunded` | 환불 |

---

## E. 채팅 리스트 상태 배지 매핑

| 배지 | 조건(해당 작가-고객 예약 중 가장 진행된 단계) |
|------|-----------------------------------------------|
| **촬영 완료** | `shot` / `delivered` / `completed` 존재 |
| **예약 완료** | `accepted` / `paid` 존재 |
| **상담 중** | 그 외(요청중·예약 없음 등) |

---

## F. 주요 엣지 케이스

- **본인 작가 사진/프로필에서 예약 시도** → `/studio` 로 리다이렉트(자기 자신과 예약 불가).
- **시간 충돌** → 작가 수락 거부(겹치는 점유 예약 존재 시).
- **비로그인 보호 동작** → `/login?next=…` 후 복귀.
- **미승인 작가** → 탐색/공개에서 RLS로 제외.
- **빈 대화방** → 채팅 리스트에서 숨김(메시지 오가야 노출).
- **나가기 후 새 메시지** → 자동으로 다시 표시.
- **상담 정보** → 고객만 작성/수정, 작가는 열람만(RLS).
