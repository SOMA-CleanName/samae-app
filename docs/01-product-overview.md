# 01. 서비스 기능 정의

## 한 줄 정의

> **samae** — 사진작가를 탐색하고 채팅으로 협의하고, 예약·결제부터 보정본 수령까지 한 흐름에서 끝내는 사진작가 매칭 마켓플레이스.

## 기능 흐름

### 유저 흐름 (해피 패스)
```
탐색(갤러리/필터) → 사진·작가 선택 → 작가 프로필 열람
→ 채팅으로 컨셉·일정·장소 협의 → 작가의 예약 제안(패키지·일시·금액)
→ 결제(에스크로 보류) → 촬영 → 작가 보정본 업로드
→ 전달 확인 → 정산 확정 → 후기 작성
```

### 작가 흐름
```
가입(작가 신청) → 운영자 승인 → 프로필·활동지역·정산계좌 등록
→ 패키지 등록(가격·소요시간·보정본 수) → 포트폴리오 사진 업로드(탐색 노출)
→ 가능 시간 등록 → 문의 채팅 응대 → 예약 제안 → 촬영 → 보정본 전달 → 정산 수령
```

## v1 기능 범위

### ✅ 포함 (거래 완결형)
- 카카오 소셜 + 이메일 인증, 유저/작가/운영자 역할
- 작가 온보딩(신청→승인), 프로필·패키지·포트폴리오·가능시간 등록
- 탐색 탭(무드·지역 필터 갤러리) + 작가 공개 프로필
- 1:1 실시간 채팅(텍스트·사진), 알림
- 예약 상태머신(요청→수락→결제→촬영→전달→완료)
- 결제(PG) + 에스크로식 보류 + 전달 확인 후 작가 정산
- 보정본 업로드/만료 다운로드
- 후기·평점
- 운영자 어드민(작가 승인, 거래·정산 모니터링, 분쟁 처리)

### ❌ v1 제외 (이후 단계)
- 네이티브 모바일 앱 (반응형 웹으로 먼저)
- 작가 간 경쟁입찰·역경매
- 구독·정기권, 쿠폰·프로모션 엔진
- AI 추천(취향 기반 자동 매칭) — v1은 필터·검색 중심
- 다국어(영문) — v1은 한국어 우선
- 정산 자동 세금계산서·원천징수 자동화 (수기 운영으로 시작)

## 비기능 요구사항

- **반응형 웹** — 모바일 우선
- **결제 신뢰성** — 결제·정산은 멱등(idempotent)·감사로그 필수
- **개인정보** — 정산계좌·연락처 등 민감정보 암호화, 최소 수집
- **이미지 성능** — 포트폴리오/보정본 다량 이미지 최적화(썸네일·지연로딩)
