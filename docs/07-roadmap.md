# 07. 개발 로드맵

반응형 웹 우선, 거래 완결형 v1. 솔로 기준 대략 2~3개월. 각 단계는 **독립적으로 동작 가능한 산출물**을 남긴다.

---

## 0단계 · 기반 (~1주)

프로젝트와 데이터 토대.

- [ ] Next.js 16 + TypeScript + Tailwind v4 스캐폴드 (`samae-app`)
- [ ] Supabase 프로젝트 생성, 환경변수 연결
- [ ] **마이그레이션 `0001_init.sql`** — [04 스키마](04-database-schema.md) 전체(enum→테이블→인덱스→RLS→트리거)
- [ ] Supabase Auth: 카카오 소셜 + 이메일, `profiles` 자동생성 트리거
- [ ] 인증 미들웨어·세션(`@supabase/ssr`), role 분기
- [ ] 랜딩 디자인 토큰 이식(색·폰트·타이포), 공통 레이아웃/헤더
- [ ] 라우트 그룹 골격 ((auth)/(user)/(photographer)/(admin)/api)

**완료 기준**: 로그인/로그아웃 동작, 빈 탐색 홈·작가 대시보드·어드민 셸이 role에 맞게 열림.

## 1단계 · 공급(작가) (~1~2주)

판매자가 진짜로 존재하게.

- [ ] 작가 신청 플로우 (유저 → `photographers` pending)
- [ ] 운영자 작가 승인 (어드민)
- [ ] 작가 대시보드: 프로필·활동지역·무드태그·정산계좌
- [ ] 패키지 CRUD
- [ ] 포트폴리오 사진 업로드(서명 URL + `sharp` 썸네일) → `photos`
- [ ] 가능 시간(`availability`) 등록

**완료 기준**: 승인된 작가가 프로필·패키지·포트폴리오를 갖춘 상태로 존재.

## 2단계 · 탐색 (~1주)

수요자가 작가를 발견하게. *(랜딩 디자인·구조 참고 지점)*

- [ ] 탐색 홈 갤러리(published photos) — 무드·지역 필터
- [ ] 작가 공개 프로필(`/photographers/[handle]`) — 인스타식 포트폴리오 그리드, 패키지·평점
- [ ] 찜(`favorites`)
- [ ] 검색(이름·지역)

**완료 기준**: 비로그인/유저가 탐색→작가 프로필까지 매끄럽게 도달.

## 3단계 · 채팅 (~1~2주)

협의 채널.

- [ ] 대화방 생성(작가 프로필 "채팅하기")
- [ ] 실시간 메시지(Supabase Realtime), 텍스트·사진
- [ ] 읽음·안읽음, 대화 목록
- [ ] 인앱 알림(`notifications`) 기본

**완료 기준**: 유저↔작가가 실시간으로 대화·사진 공유.

## 4단계 · 예약 (~1~2주)

거래 골격.

- [ ] 예약 요청(유저) → 작가 수락/거절
- [ ] 예약 상태머신 서버 라우트/RPC ([05](05-booking-lifecycle.md)), 클라이언트 직접 전이 차단
- [ ] 작가 가능시간 연동·슬롯 보류
- [ ] 양측 예약 목록·상세, 알림

**완료 기준**: 결제 직전(`accepted`)까지 예약 흐름 완결.

## 5단계 · 결제·정산 (~2~3주, 규제 검토 동반) ⚠️

가장 무거운 단계. [06](06-payment-settlement.md) 선행 결정 필요.

- [ ] PG 선정·가맹/정산 자격
- [ ] 결제 사전등록 + 결제 + **webhook(서명검증·멱등·금액대조)**
- [ ] `accepted → paid` 전이 + 에스크로 보류
- [ ] 정산 확정(`completed` 시) + 환불
- [ ] 운영자 정산·분쟁 UI, 감사로그
- [ ] 약관·환불정책·개인정보 고지

**완료 기준**: 실제 결제→보류→정산→환불 전 구간 동작(테스트 PG 포함).

## 6단계 · 전달·후기 (~1주)

거래 마무리.

- [ ] 보정본 업로드(작가) → `deliveries`, 만료 다운로드 링크
- [ ] 유저 전달 확인 → `completed` → 정산 트리거
- [ ] 자동 확인(N일) 배치
- [ ] 후기·평점, 작가 집계 갱신

**완료 기준**: 탐색→채팅→예약→결제→보정본→정산→후기 **전 구간 E2E** 완주.

## 의존성·선행 결정

- **5단계 전**: PG 선정·정산 구조·약관 (법무/세무) — 일찍 시작할수록 좋음
- **0단계 전 (외부 셋업 — 사용자 준비물)**:
  - samae 전용 **새 Supabase 프로젝트** 생성 → URL·anon·service_role 키
  - **카카오 OAuth 앱** 등록(카카오 개발자) → REST API 키·시크릿, Redirect URI를 Supabase Auth에 연결
  - 본 문서 세트 확정 (← 현재 지점)

> 결정 완료(2026-05-29): role 모델 = `user|admin` + 작가는 `photographers` 행으로 판단 / Supabase 새 프로젝트 / 인증은 카카오 소셜 포함.
