# 06. 결제·수수료

> ⚠️ **이 문서는 기술 설계 + 리스크 정리이며 법률 자문이 아니다.**
> **모델 변경(2026-06-02):** PG 에스크로/정산 분배 구조를 폐기하고, **사용자가 작가에게
> 촬영비를 직접 계좌이체**하는 방식으로 전환했다. 플랫폼은 자금을 보관·이체하지 않으며,
> **매칭 건당 정액 수수료(작가 부담)** 만 받는다. (구 PG 정산 설계 이력은 git 참조)

## 핵심 원칙

- **결제 = 사용자 → 작가 직접 계좌이체.** 플랫폼은 자금 흐름에 개입하지 않는다.
  → 전자금융거래법상 자금이체/에스크로 이슈를 회피하고 PG 가맹·정산 부담을 없앤다.
- **플랫폼 매출 = 매칭 건당 정액 수수료 6,000원.** 사용자가 아니라 **작가가 부담**한다(숨고식 리드 수수료).
- **수수료 발생 시점 = 입금 확인.** 작가가 입금을 확인(`accepted → paid`)하는 순간 수수료가 발생(accrued)하고, **월 단위로 누적·청구**한다.

상수: `PLATFORM_FEE_KRW = 6000` (`src/lib/payments.ts`).

## 자금 흐름

```
사용자 ──(촬영비 전액 직접 계좌이체)──► 작가
                                         │
                          작가가 "입금 확인" (accepted → paid)
                                         │
                                         ▼
                    플랫폼 수수료 6,000원 발생(accrued) ─누적─► 월 청구(billed) ─► 작가 납부(paid)
```

- **촬영비**: 플랫폼을 거치지 않음. `payments` 행은 *입금 확인 기록*일 뿐(보관 X).
- **수수료**: `platform_fees` 원장에 작가별로 누적. 플랫폼이 작가에게 청구.

## 상태 흐름 ([05](05-booking-lifecycle.md)와 연동)

| 시점 | bookings.status | 행위자 | 시스템 처리 |
|------|-----------------|--------|-------------|
| 작가 수락 | `accepted` | 작가 | 사용자에게 작가 수취 계좌 노출(송금 안내) |
| 직접 송금 | (오프플랫폼) | 사용자 | — (시스템 밖) |
| 입금 확인 | `accepted → paid` | 작가 | `payments=paid` + `platform_fees=accrued`(6,000원) |
| 촬영/전달 | `shot`/`delivered` | 작가 | 상태 전이 |
| 거래 완료 | `delivered → completed` | 사용자 | 완료 기록(정산 트리거 없음 — 작가는 이미 수령) |

## 데이터 모델 (마이그레이션 `0007_direct_transfer.sql`)

- **`payout_accounts`** — 작가 수취 계좌(은행/계좌번호/예금주). 작가당 1행.
  - RLS: 소유 작가/운영자만 직접 조회. **구매자에게는 서버(service_role) 경유로만 노출** — 해당 예약 참여자인지 검증 후 `getPayoutAccountForBooking()`이 반환.
  - 구 `photographers.settlement_account` 컬럼은 제거.
- **`platform_fees`** — 작가가 낼 매칭 수수료 원장. 예약 1:1.
  - `fee_krw`(기본 6000), `status`(accrued→billed→paid / waived), `period`('YYYY-MM').
  - RLS: 작가 본인/운영자 조회.
- **`payments`** — 직접이체 *확인 기록*(보관 아님). `provider/method='bank_transfer'`, `amount_krw`=촬영비, `paid_at`=작가 확인 시각.

## 환불

- 촬영비 환불 송금은 **작가가 직접 사용자에게** 한다(오프플랫폼).
- 시스템은 상태만 정리(`refundBooking`): `bookings=refunded` + `payments=refunded` + **`platform_fees=waived`(수수료 면제)** + 예약 슬롯 해제 + 양측 알림.
- 부분환불은 당사자 간 처리(시스템은 전액 환불 기준만 기록).

## 보안 포인트

- **작가 계좌 노출 게이트**: `payout_accounts`는 소유자만 RLS 조회 가능. 구매자 노출은 "해당 예약이 RLS로 보이는 참여자인가"를 먼저 확인한 뒤 admin 클라이언트로 계좌를 읽어 반환한다. 비참여자·비로그인은 계좌를 얻을 수 없다.
- 모든 상태/원장 write는 `service_role`(admin)로만. 멱등: `payments`/`platform_fees`는 `booking_id` unique.
