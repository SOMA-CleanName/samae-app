## 심각도 정의
- **치명**: 데이터 유실·권한 누수·개인정보 조기 노출·결제/입금 게이팅 우회 등 즉시 수정
- **높음**: 핵심 플로우 동작 불가/오류, 관계형 무결성 깨짐
- **중간**: 일부 상황에서 오류, 미래 리스크
- **낮음**: 죽은 코드, 정리·개선 권고

## 수정 원칙
- 두 에이전트는 감사·문서화만(소스 무수정). 실제 수정은 메인 세션이 청크 단위로 검토하며 수행.
- 마이그레이션 필요한 수정은 SQL을 별도로 제공(운영 DB는 사용자가 직접 적용).
- 커밋/푸시/PR는 사용자 명시 지시 후에만.
